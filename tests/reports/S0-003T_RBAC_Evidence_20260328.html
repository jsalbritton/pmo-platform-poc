Security Observations

OBS-001 · ml_feature_requests INSERT policy declares roles={public} — recommend {authenticated}

The INSERT policy on ml_feature_requests uses roles={public} rather than roles={authenticated}. The WITH CHECK expression correctly blocks unauthenticated access since rls_user_role() returns NULL for anon. However, defense-in-depth best practice is to also declare roles={authenticated} to reject unauthenticated attempts before the WITH CHECK is evaluated. Security posture is functionally correct. Recommend patching before Sprint 2A production build.

OBS-002 · Test user accounts (PmoTest2026!) must be deactivated before production handoff

Six test users were provisioned with the shared test password PmoTest2026!: kai.young (admin), drew.green (pm), hollis.mitchell (sponsor), blake.turner (team_member), avery.williams (viewer), ml.admin (ml_admin). These credentials are documented in tests/test_rbac_s0_003t.py. These accounts must have passwords rotated or be deactivated before the consulting firm receives the codebase. Track as a production checklist pre-condition.

OBS-003 · ml_model_versions write path deferred to S2A-007T (ML Admin UI not yet built)

The ml_model_versions write RLS policy (admin + ml_admin only, ALL operations) was validated at the SQL layer in Sprint 0. REST API path testing for this policy will be added to S2A-007T when the model version timeline UI is implemented in Sprint 2A. The RLS policy is in place and enforced. This is a test coverage gap, not a security gap.
